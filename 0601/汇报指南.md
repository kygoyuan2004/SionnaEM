## 2. 汇报时需要强调

### 不要逐行念参数表

参数表是支撑材料，不是汇报主线。汇报时应围绕两个问题展开：

- 哪些参数决定 MDS 的形态？
- 哪些参数决定 Sionna RT 仿真的可信度和可复现性？

### 微多普勒主线要讲清楚

最重要的链路是：

```
载频 f_c / 波长 λ
    -> 调制指数 β_m
    -> 峰值频偏 f_dev_peak
    -> STFT 频率范围与 MDS 清晰度
```

解释时可以说：

“28 GHz 相比 3.5 GHz 波长更短，因此同样的叶片半径和旋转速度会产生更大的相位调制指数，微多普勒谱更明显；但频率越高路径损耗也越大，所以这是一个检测能力和覆盖距离之间的 trade-off。”

### 旋翼参数要和频谱结果绑定

`R` 和 `f_rot` 不只是几何/机械参数，它们共同决定叶片尖端速度和微多普勒带宽：

- `R` 越大，叶片尖端运动半径越大。
- `f_rot` 越大，旋翼转得越快。
- 二者共同增大 `f_dev_peak`，使 MDS 展开更宽。

### Sionna RT 参数要讲成数据生成链路

不要把 `PathSolver`、`paths.cir()`、RadioMap、CSI 当成孤立 API。推荐讲成：

```
场景与设备参数
    -> PathSolver 计算传播路径
    -> paths.cir() 得到 CIR
    -> CFR / CSI / RadioMap 支撑后续分析
```

这样老师更容易理解：代码参数不是为了罗列接口，而是为了把物理场景转成可分析数据。

## 3. 老师可能追问的问题

| 可能问题 | 回答要点 |
|----------|----------|
| 为什么 28 GHz 比 3.5 GHz 微多普勒更明显？ | 微多普勒相位调制与 `1 / λ` 或 `f_c` 成正相关。28 GHz 波长更短，所以同样旋翼运动会产生更强的调制。但高频路径损耗更大，覆盖距离更短。 |
| STFT 窗长为什么有 trade-off？ | 窗长增大，频率分辨率变好，但时间分辨率变差；窗长减小，时间变化更敏感，但频率分辨率变差。MDS 需要在“看清边带”和“跟踪时间变化”之间折中。 |
| `max_depth`、`diffuse_reflection`、`samples_per_src` 分别影响什么？ | `max_depth` 控制反射/透射/散射路径复杂度；`diffuse_reflection` 决定是否考虑漫反射；`samples_per_src` 控制蒙特卡洛采样量，越大越稳定但计算更慢。 |
| `参数.pdf` 和 `参数_code.pdf` 的关系是什么？ | `参数.pdf` 讲项目参数体系和物理意义；`参数_code.pdf` 讲这些参数在 Sionna RT 代码接口中的配置位置。前者回答“为什么”，后者回答“怎么落地”。 |
| 为什么不能只看代码参数？ | 代码参数必须回到物理意义解释，否则无法判断哪些参数影响 MDS、哪些参数只是仿真精度或显示配置。 |
